# Application Process

## Purpose

This page summarizes how a project enters the OSL workflow.

## Scope

Use this process for POC, incubation, graduation, and promotion
requests.

## Steps

1. Open an issue with the matching template.
2. State the current stage and the requested stage.
3. Link the project repository and supporting evidence.
4. Identify the ecosystem context if relevant.
5. Wait for reviewer assignment.
6. Respond to feedback and update the project.

## Rule

The issue should contain enough evidence that a reviewer can start
checking the correct checklist without asking for basic setup details.
