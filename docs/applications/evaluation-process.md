# Evaluation Process

## Purpose

This page describes how a submitted project is reviewed.

## Scope

Use it for checklist review, due diligence, and stage decisions.

## Steps

1. A project applies via issue.
2. An editor assigns or confirms a reviewer.
3. The reviewer checks the evidence against the checklist.
4. The reviewer posts concise feedback and follow-up items.
5. The author updates the project.
6. The reviewer rechecks the gaps.
7. The editor records the decision.

## Rules

- All reviews are async.
- All feedback must be tied to checklist items or due-diligence notes.
- Reviewers should not block on taste or preference.
- Editors own the final stage transition decision.
