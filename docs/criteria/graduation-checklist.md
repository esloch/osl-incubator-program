# Graduation Checklist

## Purpose

Use this checklist to confirm a project is ready for the adoption and
maturity stage.

## Scope

This stage checks that the project can sustain itself with visible use
and a maintainable contributor model.

## Required

- [ ] Stable release history or equivalent change record
- [ ] Multiple contributors or a credible multi-contributor path
- [ ] Active maintenance
- [ ] Mature documentation for users and contributors
- [ ] Governance model is defined
- [ ] Security policy exists

## Adoption signals

- [ ] External users or adopters are visible
- [ ] Releases happen regularly enough to be credible
- [ ] Issues are answered in a reasonable time
- [ ] Maintenance tasks are not dependent on a single person

## Ecosystem note

If the project is headed toward a specific ecosystem review, use that
ecosystem's deeper guidance as a follow-up reference.
