# Python Guidance

## Purpose

This page describes the baseline OSL expectations for Python projects.

## Scope

Use it for scientific Python, libraries, and tools that may later be
validated more deeply by pyOpenSci.

## OSL baseline

- the project is packaged in a standard way
- installation is documented
- basic usage is documented
- tests exist for important behavior
- the repository can be reviewed without guessing the runtime setup

## Delegated depth

pyOpenSci should be used for deeper Python-specific validation when a
project is ready for that level of review.

OSL should not recreate those detailed requirements here.

## Reviewer focus

- packaging is understandable
- documentation is sufficient for contributors and users
- tests protect the core behavior
- the project can be installed or exercised reliably
