# R Guidance

## Purpose

This page describes the baseline OSL expectations for R projects.

## Scope

Use it for R packages and tools that may later be validated more deeply
by rOpenSci.

## OSL baseline

- installation is documented
- user-facing documentation exists
- tests exist for important behavior
- the release or build path is understandable
- the repository can be reviewed without hidden setup steps

## Delegated depth

rOpenSci should be used for deeper R-specific validation when a project
is ready for that level of review.

OSL should not duplicate those detailed requirements here.

## Reviewer focus

- documentation is clear and current
- the project is maintainable
- packaging or release steps are understandable
- evidence supports the project claim
