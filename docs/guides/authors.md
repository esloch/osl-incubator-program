# Authors

## Purpose

Authors build the project and keep it moving through the OSL workflow.

## Scope

Use this page if you are submitting a project or responding to review
feedback.

## Responsibilities

- prepare the application issue
- provide accurate evidence
- keep the repository current
- respond to review feedback
- update the project until the checklist is satisfied

## Author rule

If a reviewer asks for follow-up, respond with a concrete change or a
clear explanation of why the item does not apply.

## Good evidence

- links to docs
- links to releases
- test results
- demo or deployment evidence
