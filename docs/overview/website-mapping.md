# Website Mapping

## Purpose

This page defines the relationship between the OSL website and this
repository.

## Scope

Use it whenever someone asks where a policy, checklist, or workflow
should live.

## Rule

- The website is the entry point for readers.
- This repository is the source of truth for program content.

## What belongs where

### Website

The website should link to the program pages and explain how to start.

It should not become a second copy of the workflow or criteria.

### Repository

The repository should contain the canonical markdown for:

- stages
- criteria
- workflows
- role guidance
- ecosystem guidance
- templates

## Maintenance rule

If a process changes, update the repository first and then update the
website links or summaries to match.
