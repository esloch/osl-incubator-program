# Incubation Stage

## Purpose

Incubation is the stabilization phase for projects that have moved past
proof of concept.

## Scope

Use incubation when the project needs repeatable maintenance,
documentation, governance, and reviewable release practices.

## Focus

- stable installation and usage
- docs that support contributors and users
- lightweight governance
- test coverage for important paths
- repeatable release or deployment steps

## What reviewers look for

- the project follows its own process
- the team can make regular improvements without relying on a single
  person
- the project can be reproduced or deployed in a predictable way
- the checklist evidence is current and easy to verify

## Exit expectation

The project should be usable, reviewed, and stable enough for broader
adoption.
